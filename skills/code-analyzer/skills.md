---
name: code-analyzer
description: >
  Your task is to analyze the provided software project (codebase / description / features) and generate a complete, professional Software Specification Document (SSD).
---
Follow these instructions carefully:

1. Understand the system:
   - Identify the purpose of the application
   - Determine the domain (e.g., finance, healthcare, workflow management, etc.)
   - Identify key modules, workflows, and actors

2. Generate a structured document with the following sections:

### 1. Introduction
- Purpose of the system
- Scope of the document
- Definitions, acronyms, abbreviations

### 2. Overall Description
- Product perspective
- Product functions (high-level features)
- User classes and characteristics
- Assumptions and dependencies

### 3. System Architecture
- High-level architecture (e.g., monolith, microservices)
- Technology stack (backend, frontend, database, APIs)
- Integration points

### 4. Functional Requirements
- Break down features module-wise
- Use clear, numbered requirements (FR-1, FR-2, etc.)
- Include inputs, processing, outputs

### 5. Non-Functional Requirements
- Performance
- Security
- Scalability
- Availability
- Maintainability

### 6. Data Model
- Key entities and relationships
- Database schema (tables, fields)
- Sample JSON structures if applicable

### 7. API Specifications (if applicable)
- Endpoint list
- Request/response formats
- Authentication methods

### 8. User Interface Overview
- Key screens/pages
- User workflows

### 9. Business Rules
- Important logic and constraints

### 10. Error Handling & Edge Cases
- Possible failures and system behavior

### 11. Future Enhancements
- Suggested improvements or extensions

### 12. Appendix
- Any additional technical details


3. Output Guidelines:
- Use clear headings and structured formatting
- Be concise but comprehensive
- Use tables where appropriate
- Avoid assumptions—state them explicitly if needed
